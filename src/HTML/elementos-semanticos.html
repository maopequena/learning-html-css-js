Elementos semânticos

São elementos que exibem seu significado claramente para o navegador e o desenvolvedor. Os elementos semânticos proporcionam uma Web semântica, o que possibilita que os dados sejam compartilhados e reutilizados entre aplicativos, empresas e comunidades.

P. ex. <div> e <span> não informam qual é seu conteúdo; no entanto <form>, <table> e <article>, sim.

Por que usar elementos semânticos?

  • Acessibilidade: um HTML semântico torna as páginas da Web acessíveis para dispositivos móveis e pessoas com deficiências, porque os leitores de tela e navegadores conseguem interpretar melhor o código.
  • SEO: eles melhoram o SEO, ou Search Engine Optimization (Otimização para mecanismos de pesquisa), do site, que é o processo de aumentar a quantidade de pessoas que visita sua página. Com um SEO melhor, os mecanismos de pesquisa conseguem identificar melhor o conteúdo do seu site e avaliar o conteúdo mais importante apropriadamente.
  • Facilidade de compreensão: o HTML semântico também torna o código-fonte do site mais fácil de ler para outros desenvolvedores.

Para compreender melhor, é como entrar num supermercado que não identifica seus corredores: fica difícil encontrar os produtos. Porém, quando um supermercado identifica seus corredores, é bem mais fácil achar o que se procura.

Uso do elemento "section"

O elemento <section> define uma seção em um documento, que pode ser definida como "um agrupamento temático de conteúdo, geralmente com um cabeçalho".

Exemplos de uso para o elemento <section>:

  • Capítulos
  • Introdução
  • Informações de contato
  • Novos itens

Uso do elemento "article"

O elemento <article> define um conteúdo independente e autocontido. O que está contido nesta tag deve fazer sentido por si só, e deve poder ser distribuído independentemente do restante do site.

Exemplos de uso para este elemento:

  • Postagens em fóruns
  • Postagens em blogs
  • Comentários de usuários
  • Cartões de produtos
  • Artigos de jornais e revistas

Elemento <header>

Esse elemento representa um contêiner para um conteúdo introdutório ou um conjunto de links de navegação.

Um elemento <header> geralmente contém:

  • Um ou mais elementos de cabeçalho (<h1>-<h6>)
  • Um logotipo ou um ícone
  • Informações sobre o autor

Pode haver vários elementos <header> em um documento HTML, porém ele NÃO pode ser aninhado nos seguintes elementos: <footer>, <address> ou outro <header>.

Uso do elemento "nav"

Esse elemento define um conjunto de links de navegação. NEM TODOS os links de um documento devem ser colocados no elemento <nav>. Ele deve ser usado apenas para os principais blocos de links de navegação. Navegadores e leitores de tela para usuários com alguma deficiência podem usar este elemento para determinar se devem omitir o processamento inicial deste conteúdo.

Uso do elemento "aside"

Esse elemento define um conteúdo ao lado do conteúdo em que está aninhado, como uma barra lateral. O conteúdo de <aside> deve estar indiretamente relacionado ao conteúdo que o rodeia.

Ele é usado para definir informações adicionais que podem aprimorar o conhecimento sobre outro elemento, mas não é obrigatório para compreender o conteúdo principal.

Usos comuns para o elemento <aside>:

  • Bibliografias
  • Notas de rodapé
  • Comentários
  • Citações extraídas do texto
  • Informações adicionais sobre o texto ou a publicação
  • Mais informações

Uso dos elementos "figure" e "figcaption"

A tag <figure> define um conteúdo autocontido, como ilustrações, diagramas, fotos, listagens de código, etc. A tag <figcaption> define uma legenda para um elemento <figure> e pode ser colocado como o primeiro ou o último filho de um elemento <figure>. O elemento <img> define a imagem/ilustração em si.

Uso do elemento "footer"

Esse elemento define o rodapé de um documento ou uma seção.

Um elemento <footer> geralmente contém:

  • Informações sobre o autor
  • Informações de copyright
  • Informações de contato
  • Termos de uso e avisos de privacidade
  • Um mapa do site
  • Links para voltar ao topo da página
  • Documentos relacionados

Pode haver vários elementos <footer> em um documento HTML.
